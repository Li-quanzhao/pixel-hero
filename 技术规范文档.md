## 一、技术栈

### 1.1 核心技术

| 分类 | 技术 | 版本 | 用途 |

| 模块 | 用途 | 实际使用状态 |
|------|------|-------------|

### 1.3 编译器说明

实际编译使用独立安装的 MinGW-w64 16.1.0：

```
g++.exe (x86_64-win32-seh-rev1, Built by MinGW-Builds project) 16.1.0
```

Qt 6.11.1 自带的配套 MinGW 版本为 **13.1.0**（位于 `E:\QT\Tools\mingw1310_64\bin`），两个版本均可用于编译，但运行时 DLL 必须与编译时使用的版本一致。

## 二、编码规范

### 2.1 命名规范

#### 类命名
- **规则**: PascalCase（大驼峰）
- **示例**: `GameWindow`, `MapManager`, `BattleSystem`

#### 方法命名
- **规则**: camelCase（小驼峰）
- **示例**: `startGame()`, `loadMap()`, `takeDamage()`

#### 成员变量命名
- **规则**: `m_` 前缀 + camelCase
- **示例**: `m_player`, `m_isPaused`, `m_currentMap`

#### 常量命名
- **规则**: 全大写 + 下划线分隔
- **示例**: `MAX_HEALTH`, `DEFAULT_SPEED`

#### 枚举命名
- **枚举类型**: PascalCase
- **枚举值**: 全大写 + 下划线分隔
```cpp
enum class TileType {
    FLOOR,
    WALL,
    WATER
};
```

### 2.2 文件组织规范

#### 头文件结构
```cpp
#ifndef CLASSNAME_H
#define CLASSNAME_H

// 1. Qt头文件
#include <QGraphicsItem>
#include <QString>

// 2. 项目头文件
#include "Player.h"

// 3. 前向声明
class Enemy;

// 4. 类定义
class ClassName : public BaseClass
{
    Q_OBJECT  // 如果是QObject子类

public:
    // 构造函数
    ClassName(QObject *parent = nullptr);
    ~ClassName();

    // 公有方法
    void publicMethod();

signals:
    // Qt信号

public slots:
    // Qt槽函数

private:
    // 私有方法
    void privateMethod();

    // 成员变量
    int m_value;
};

#endif // CLASSNAME_H
```

#### 源文件结构
```cpp
#include "ClassName.h"
// 其他头文件

// 构造函数实现
ClassName::ClassName(QObject *parent)
    : BaseClass(parent), m_value(0)
{
}

// 析构函数实现
ClassName::~ClassName()
{
}

// 方法实现（按头文件声明顺序）
void ClassName::publicMethod()
{
}
```

### 2.3 代码风格

#### 缩进与空格
- 缩进：4个空格
- 大括号：Allman风格（换行）
```cpp
if (condition)
{
    // code
}
```

#### 指针与引用
- 指针星号靠近类型：`Player* player`
- 引用符号靠近类型：`const QString& name`

#### 包含保护
- 使用 `#ifndef` / `#define` / `#endif`
- 宏名称格式：`FILENAME_H`

### 4.2 QGraphicsItem 方法重写注意事项

**重要**: `QGraphicsItem::type()` 返回 `int`，子类不应定义同名方法返回其他类型。

```cpp
// 错误示例
class Tile : public QGraphicsPixmapItem {
    TileType type() const;  // 与父类方法冲突！
};

// 正确示例
class Tile : public QGraphicsPixmapItem {
    TileType tileType() const;  // 使用不同名称
};
```

### 4.3 QObject 信号槽规范

- 信号命名：使用过去式或状态变化描述
```cpp
signals:
    void gameStarted();    // 正确
    void gameOvered();     // 避免与同名方法冲突
```

- 连接方式：使用新式连接
```cpp
connect(timer, &QTimer::timeout, this, &GameScene::updateGame);
```

## 五、构建规范

### 5.1 环境要求

| 项目 | 要求 |
|------|------|
| Qt版本 | 6.11.1 (mingw_64) |
| Qt安装路径 | `E:\QT\6.11.1\mingw_64` |
| 编译器 | MinGW 16.1.0 (或 Qt 配套的 13.1.0) |
| MinGW路径 | `E:\MinGW-w64\mingw64` (或 `E:\QT\Tools\mingw1310_64`) |
| C++标准 | C++17 |
| 编码 | UTF-8 |

### 5.2 构建命令

```powershell
# 设置环境变量（使用独立MinGW 16.1.0）
$env:PATH = "E:\MinGW-w64\mingw64\bin;E:\QT\6.11.1\mingw_64\bin;" + $env:PATH

# 设置环境变量（使用Qt配套MinGW 13.1.0）
$env:PATH = "E:\QT\Tools\mingw1310_64\bin;E:\QT\6.11.1\mingw_64\bin;" + $env:PATH

# 进入项目目录
cd pixel_hero

# 1. 生成Makefile
qmake pixel_hero.pro

# 2. 编译（并行4线程）
mingw32-make -j4

# 3. 清理
mingw32-make clean

# 4. 完全重新编译
mingw32-make clean
qmake pixel_hero.pro
mingw32-make -j4
```

### 5.3 运行时部署

编译后需将运行时库部署到 `bin\` 目录，才能脱离开发环境独立运行：

```powershell
# Qt运行时库（从Qt安装目录复制）
copy E:\QT\6.11.1\mingw_64\bin\Qt6Core.dll      bin\
copy E:\QT\6.11.1\mingw_64\bin\Qt6Gui.dll        bin\
copy E:\QT\6.11.1\mingw_64\bin\Qt6Widgets.dll    bin\

# MinGW运行时库（从MinGW安装目录复制）
copy E:\MinGW-w64\mingw64\bin\libgcc_s_seh-1.dll    bin\
copy E:\MinGW-w64\mingw64\bin\libstdc++-6.dll       bin\
copy E:\MinGW-w64\mingw64\bin\libwinpthread-1.dll   bin\

# Qt平台插件
mkdir bin\platforms
copy E:\QT\6.11.1\mingw_64\plugins\platforms\qwindows.dll bin\platforms\

# Qt图片格式插件（可选）
mkdir bin\imageformats
copy E:\QT\6.11.1\mingw_64\plugins\imageformats\*.dll bin\imageformats\
```

### 5.4 编译兼容性说明

使用独立 MinGW 16.1.0 编译时需要手动修改 Makefile.Release：

**修改1**：移除 `QT_NEEDS_QMAIN` 定义
```makefile
# 原
DEFINES = ... -DQT_NEEDS_QMAIN
# 改
DEFINES = ...  # 删除 -DQT_NEEDS_QMAIN
```

**修改2**：移除 `libQt6EntryPoint.a` 链接
```makefile
# 原
LIBS = ... -lmingw32 E:\QT\6.11.1\mingw_64\lib\libQt6EntryPoint.a -lshell32
# 改
LIBS = ... -lshell32  # 删除 -lmingw32 和 libQt6EntryPoint.a
```

如果使用 Qt 配套的 MinGW 13.1.0，则无需上述修改。

## 十、性能优化建议

### 10.1 渲染优化

- 使用 `QGraphicsItem::setCacheMode()` 缓存静态元素
- 避免频繁的 `addItem()` / `removeItem()` 操作
- 使用对象池复用游戏对象

### 10.2 碰撞检测优化

- 使用空间分区减少检测次数
- AABB包围盒预计算
- 只检测可见区域内的碰撞
