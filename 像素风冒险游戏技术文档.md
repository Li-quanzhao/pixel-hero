## 一、游戏概念与核心玩法

### 1.1 游戏名称
**《像素勇者》** - Pixel Hero Adventure

### 1.2 游戏类型
2D像素风动作冒险游戏

### 1.3 核心玩法
| 玩法 | 描述 |
|------|------|
| **探索** | 玩家控制角色在像素风格的世界中探索多个区域 |
| **战斗** | 与敌人进行实时战斗，支持普通攻击和技能释放 |
| **收集** | 收集道具、武器、装备提升角色能力 |
| **解谜** | 解开环境谜题推进剧情发展 |

### 1.4 游戏特色
- 复古像素艺术风格
- 流畅的实时战斗系统
- 丰富的角色成长系统
- 多样化的关卡设计

### 1.5 游戏玩法流程
```
开始游戏 → 创建角色 → 新手村 → 接受任务 → 探索地图 → 战斗 → 收集装备 → 挑战Boss → 通关
```

## 二、需求分析

### 2.1 核心功能需求

| 功能模块 | 需求描述 | 优先级 |
|---------|---------|--------|
| **角色系统** | 玩家角色创建、属性成长、技能系统 | 高 |
| **战斗系统** | 实时战斗、攻击/防御/技能释放 | 高 |
| **地图系统** | 像素地图渲染、场景切换、碰撞检测 | 高 |
| **道具系统** | 道具收集、装备管理、背包系统 | 中 |
| **NPC系统** | NPC对话、任务系统、商店系统 | 中 |
| **存档系统** | 游戏进度保存与读取 | 中 |
| **音效系统** | 背景音乐、战斗音效、环境音效 | 低 |

### 2.2 非功能需求
- 支持Windows/macOS/Linux跨平台运行
- 游戏帧率稳定在60fps
- 响应时间<100ms

## 六、数据设计

### 6.1 JSON数据文件

游戏数据以JSON格式存储，通过Qt资源系统编译到可执行文件中。

#### 配置文件格式 - levels.json

```json
{
  "levels": [
    {
      "id": "level_1",
      "name": "新手村",
      "width": 32,
      "height": 24,
      "tileset": "village.png",
      "spawn_point": {"x": 16, "y": 12},
      "enemies": ["goblin", "slime"],
      "npcs": ["blacksmith", "shopkeeper"]
    }
  ]
}
```

#### 配置文件格式 - items.json

```json
{
  "items": [
    {
      "id": "sword_001",
      "name": "铁剑",
      "type": "weapon",
      "attack": 10,
      "price": 100,
      "description": "普通的铁剑"
    }
  ]
}
```

### 6.2 资源文件配置 (resources.qrc)

```xml
<RCC>
    <qresource prefix="/">
        <file>resources/data/levels.json</file>
        <file>resources/data/items.json</file>
    </qresource>
</RCC>
```

### 6.3 存档数据结构 (SaveManager)

存档使用JSON序列化，以文件形式存储在本地：

```cpp
struct SaveData {
    QString playerName;     // 玩家名称

    int exp;                // 经验值

    int attack;             // 攻击力
    int defense;            // 防御力
    int gold;               // 金币
    QString currentMap;     // 当前地图
    int posX, posY;         // 玩家坐标
    QStringList inventoryIds; // 背包物品ID列表
    QString weaponId;       // 武器ID
    QString armorId;        // 护甲ID
};
```

## 七、构建与部署

### 7.1 构建环境要求

| 项目 | 要求 |
|------|------|
| Qt版本 | 6.11.1 (mingw_64) |
| C++标准 | C++17 |
| 编译器 | MinGW (13.1.0 或 16.1.0) |
| 构建工具 | qmake 3.1 |

### 7.2 构建命令

```powershell
# 设置环境变量（根据实际MinGW路径调整）
$env:PATH = "E:\MinGW-w64\mingw64\bin;E:\QT\6.11.1\mingw_64\bin;" + $env:PATH

# 进入项目目录
cd pixel_hero

# 1. 生成Makefile
qmake pixel_hero.pro

# 2. 编译（Release模式）
mingw32-make -j4

# 3. 清理
mingw32-make clean
```

### 7.3 运行时部署

编译后所有文件位于 `bin\` 目录，需包含以下运行时库才能独立运行：

| 文件 | 来源 | 说明 |
|------|------|------|
| `pixel_hero.exe` | 编译产物 | 主程序 |
| `Qt6Core.dll` | `E:\QT\6.11.1\mingw_64\bin` | Qt核心运行时 |
| `Qt6Gui.dll` | `E:\QT\6.11.1\mingw_64\bin` | Qt图形运行时 |
| `Qt6Widgets.dll` | `E:\QT\6.11.1\mingw_64\bin` | Qt控件运行时 |
| `libgcc_s_seh-1.dll` | MinGW安装目录 | C++运行时 |
| `libstdc++-6.dll` | MinGW安装目录 | C++标准库 |
| `libwinpthread-1.dll` | MinGW安装目录 | 线程支持 |
| `platforms/qwindows.dll` | `E:\QT\6.11.1\mingw_64\plugins\platforms` | Windows平台插件 |
| `imageformats/q*.dll` | `E:\QT\6.11.1\mingw_64\plugins\imageformats` | 图片格式插件 |

### 7.4 常见构建问题

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| `undefined reference to __imp___argc` | 使用独立MinGW但链接了Qt EntryPoint | 移除`libQt6EntryPoint.a`和`QT_NEEDS_QMAIN`定义 |
| `-fuse-linker-plugin but liblto_plugin.dll not found` | GCC版本与编译的LTO插件不匹配 | 使用同一MinGW版本编译和链接 |
| `Could not find qmake spec 'win32-g++'` | qmake找不到mkspecs配置 | 设置正确的`Prefix`路径或复制mkspecs目录 |
| 弹窗"无法定位程序输入点" | 运行时DLL版本与编译时不一致 | 确保Qt DLL和MinGW DLL版本匹配 |

## 十、像素风格设计规范

### 10.1 颜色方案
- **主色调**: 复古像素风格，使用有限颜色调色板
- **配色原则**: 明亮鲜艳的色彩，高对比度
- **推荐色板**: 
  - 背景: #1a1a2e / #16213e
  - 地面: #4a5568 / #2d3748
  - 角色: #f5f5dc / #fffacd
  - 强调色: #e53e3e / #38a169 / #3182ce

### 10.2 像素尺寸规范

| 元素类型 | 推荐尺寸 | 说明 |
|---------|---------|------|
| 角色 | 64x64 或 48x48像素 | 保持清晰可见 |
| 地图瓦片 | 32x32像素 | 统一网格大小 |
| UI图标 | 32x32 或 16x16像素 | 根据用途选择 |
| 道具 | 32x32像素 | 背包显示用 |

### 10.3 动画帧数规范

| 动画类型 | 推荐帧数 | 说明 |
|---------|---------|------|
| 行走动画 | 4-6帧 | 流畅循环 |
| 攻击动画 | 4-8帧 | 包含预备和收招 |
| 跳跃动画 | 3-5帧 | 起跳、空中、落地 |
| 特效动画 | 6-12帧 | 技能释放效果 |

## 十二、风险预估与应对

### 12.1 风险清单

| 风险描述 | 概率 | 影响程度 | 应对措施 |
|---------|------|---------|---------|
| 游戏性能问题导致卡顿 | 中 | 高 | 使用缓存优化、减少绘制调用、对象池复用 |
| 内存泄漏 | 中 | 高 | 使用智能指针、资源池管理、定期内存检查 |
| 跨平台兼容性问题 | 低 | 中 | 使用Qt跨平台API、避免平台特定代码 |
| 测试不足导致Bug遗漏 | 中 | 中 | 编写单元测试、功能测试、QA测试 |
| 资源加载慢 | 低 | 中 | 异步加载、资源压缩、预加载机制 |
| 存档数据损坏 | 低 | 高 | 数据校验、定期备份、事务处理 |

### 12.2 性能优化策略

| 优化方向 | 具体措施 |
|---------|---------|
| 渲染优化 | 使用QGraphicsItem::cacheMode、视口裁剪、批处理绘制 |
| 内存优化 | 对象池、智能指针、及时释放无用资源 |
| 碰撞检测 | 空间分区、AABB预计算、减少检测次数 |
| 动画优化 | 帧缓存、GPU加速、避免过度绘制 |
