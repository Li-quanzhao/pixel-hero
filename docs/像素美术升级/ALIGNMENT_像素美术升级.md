## 五、边界确认

### 5.1 明确包含

- ✅ 6种敌人精灵图绘制 (史莱姆/哥布林/骷髅/蝙蝠/精英哥布林/龙)
- ✅ 3种角色精灵图绘制 (战士/弓箭手/法师)
- ✅ 4种武器图标绘制 (短剑/长剑/法杖/匕首)
- ✅ 精灵图集成到代码中 (`setPixmap` 替代 `fill(color)`)
- ✅ 使用 Python PIL/Pillow 程序化生成像素精灵图 (保证可控性)

### 5.2 明确不包含

- ❌ 动画帧 (仅静态精灵 + 简单的缩放/旋转动效)
- ❌ 场景背景 (保持纯色背景)
- ❌ 音效 + BGM
- ❌ UI 美化 (菜单/卡片已自绘，暂不改动)
- ❌ 特效粒子系统 (超出范围)

### 5.3 精灵生成方式

由于项目已使用纯色方块作为占位符，当前最务实的方案是：

**方案: Python PIL 程序化像素绘制**

- 使用 PIL/Pillow 库逐像素绘制精灵图
- 每个角色/敌人一个 `.py` 生成脚本或一个统一的 `generate_sprites.py`
- 输出 PNG 到 `pixel_hero/resources/sprites/` 目录
- 优势: 精确可控、可批量修改、无需外购素材、生成结果一致

## 七、需求理解总结

**一句话**: 使用 Python PIL 程序化绘制所有怪物/角色/武器的像素精灵图(PNG)，风格参考 Dragon Quest / Final Fantasy IV-VI 的 SNES 16-bit 像素美术，替换当前纯色方块。

**关键决策点**:
- 精灵生成方式: Python PIL 脚本 → 程序化可控
- 美术风格: SNES 16-bit JRPG 风格
- 范围: 仅精灵图，不含动画/特效/背景
- 精灵尺寸: 与当前色块一致 (角色48px，普通怪32-48px，Boss 96px)
